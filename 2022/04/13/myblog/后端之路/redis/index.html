redis

redis：内存存储（速度快）、远程操作（可以与多个客户端连接）、持久化（服务器重启可保持重启前的数据）、可扩展（主从复制、分片）

mac安装redis

✅brew install redis 安装
✅brew info redis 查看安装路径
✅启动redis：
方式一：使用brew启动 brew services start redis
方式二：进入redis的bin目录，输入redis-server
✅关闭redis服务
方式一：redis-cli SHUTDOWN
方式二：redis-cli进入再输入SHUTDOWN
✅查看redis是否在运行：ps axu|grep redis
✅客户端连接redis：redis-cli -h 127.0.0.1 -p 6379
连接后执行ping命令，检测redis服务是否启动
✅redis.conf配置文件说明：
redis默认是前台启动，如果相以守护进程（后台方式）运行，可以在配置文件中将daemonize no修改为yes。

守护进程（后台进程）与前台进程：
前台进程是在终端中运行的，终端关闭进程也就消失。
守护进程是运行在后台的一种特殊进程，不受终端控制，也不需要终端的交互。
Linux中大多数服务器是用守护进程实现的，比如web服务器http等。

常用命令

keys * 查看所有key的内容
exits key 查看key是否存在
set key value 设置key的内容
get key 获取key的内容
flushdb 删除当前数据库中所有key
flushall 删除所有数据库中key

服务器相关命令：
ping 测试连接是否存活，返回pong代表有响应
echo 输出信息
select num 选择数据库，0-15
quit 退出连接
dbsize 返回当前数据库中key的数量
info 获取服务的相关信息

设置键命令：
setex key seconds value 设置值并设置有效期，已存在会覆盖
pexpire key millisecond 设置有效时间，毫秒为单位，必须是已经存在的key
setnx key value 不存在时放入
exists key 是否存在该key
ttl key 以秒为单位显示过期时间
pttl key 以毫秒为单位显示过期时间

redis数据结构

string：可存储字符串、整数、浮点数
list：一个链表，每个节点包含一个字符串
set：包含字符串的无序收集容器
hash：包含键值的无序散列表
zset：有序集合，元素排列顺序由分值大小决定

redis hset

Redis Hset 命令用于为哈希表中的字段赋值 。
如果哈希表不存在，一个新的哈希表被创建并进行 HSET 操作。
如果字段已经存在于哈希表中，旧值将被覆盖。
hset myhashset key value

redis list相关

往list的右边加一个值
rpush mylist value
如果列表不存在则新建，如果key不是列表类型，返回错误。
lrange mylist 从左边一个一个遍历展示
llen mylist 返回长度

noauth authentication required

认证问题，需要输入密码
auth “password“

keys命令模糊查询

*通配符：keys *hh*
?单个通配符：keys hel？？ 忘了最后两个字母
[]通配符：keys hell[opt] 最后一个字母可能是o p t中的一个

redis setex命令

为指定的key设置值及其过期时间，如果已存在，则替换
setex key 10 value //10s后过期
ttl key //还有多少时间过期

misconf redis is configured to save RDB

error原因：强制关闭redis快照导致不能持久化
解决：config set stop-writes-on-bgsave-error no

Jedis和redisTemplate

Jedis是Redis官方推荐的面向Java的操作Redis的客户端，而RedisTemplate是SpringDataRedis中对JedisApi的高度封装。

SpringDataRedis相对于Jedis来说可以方便地更换Redis的Java客户端，比Jedis多了自动管理连接池的特性，方便与其他Spring框架进行搭配使用如：SpringCache
