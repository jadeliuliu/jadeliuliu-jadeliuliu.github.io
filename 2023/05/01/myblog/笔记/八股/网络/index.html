✅http是什么

HTTP (Hypertext Transter Protocol)是一种应用层协议，它是互联网上应用最广泛的一种协议之一。HTTP最初设计用于在客户端和服务器之间传输HTML页面，但随着互联网的发展，它已经被扩展到传输各种类型的数据。
HTTP协议的工作原理是客户端发送HTTP请求，请求包含了一些信息，例如请求的资源、HTTP版本、请求的方法等。服务器接收到请求后，返回HTTP响应，响应包含了一些信息，例如状态码、响应头部、响应体等。客户端收到响应后，根据响应内容进行处理。
HTTP协议是一种无状态协议，即服务器不会记录客户端的状态信息，每个请求都是独立的。为了处理客户端的状态。
HTTP 入了Cookie技术，通过在客户端存储小段数据来实现状态的保存。
由于HTTP协议是明文传输的，容易被窃听和篡改，因此现在广泛使用的是HTTPS协议，它对HTTP协议进行了加密处理，保证了数据的安全性。

✅https原理

HTTPS (Hypertext Transter Protocol Secure）是一种通过TLS/SSL加密传输数据的HTTP协议。它的原理可以简单地概括为以下几个步骤：

  1. 客户端请求建立HTTPS连接。客户端向服务器发送一个HTTPS请求，请求建立安全连接。
  2. 服务器发送数字证书。如果服务器能够建立安全连接，它会向客户端返口一个数字证书，证书包含了服务器的公钥和相关的信息（服务端选中的加密算法）。
  3. 客户端验证证书。客户端会验证数宇证书的有效性，例如证书的颁发机构是否可信，证书是否已过期等。
  4. 客户端生成会话密钥。如果数字证书有效，客户端会生成一个用于加密数据的会话密钥，并使用服务器的公钥将该密钥加密后发送给服务器。
  5. 服务器解密会活密钥。服务器使用自己的私钥解密客户端发来的会话密钥。
  6. 建立安全连接。客户端和服务器使用会话密钥（对称密钥）加密和解密数据，建立安全连接。
    使用HTTPS可以避免信息破窃取、篡改和伪造，保护用户的隐私和数据安全。
